## Latest-First Literature Sweep + New Big-Jump Candidate: Decoupled Latent Memory Bank (2026-06)

**Trigger**: After repeated falsification of embedded per-block slot designs (selection pressure, GSMU gated write, Surprise trigger, aux loss), the project diagnosed the root issue as "sparse slots tightly coupled inside every hybrid recurrence step". Per skill, further local tuning on this topology is a local minimum. The user explicitly requested "최신 논문 참고해서 해결해 봐".

**Method**: Strict latest-first sweep (2026 → 2025), focused on the precise bottleneck: how to make memory write decisions depend on actual downstream utility inside a tight recurrent latent loop (LoopLM / answer_state_loop style), avoiding high-frequency per-micro-step updates that dilute the learning signal for selectivity.

### 1. Most Relevant 2025–2026 Papers (Primary Sources Verified)

| Date | Paper | Key Mechanism | How it attacks our failure | Relevance |
|------|-------|---------------|---------------------------|---------|
| May 2026 | **MELT** (arXiv:2605.07721) "Memory-Efficient Looped Transformer: Decoupling Compute from Memory in Looped Language Models" | In LoopLM/Ouro-style recurrent latent models, replace per-iteration growing KV/memory with a **single shared evolving cache per layer + learnable gating** that controls update across loops. Enables constant-memory iterative reasoning. | Directly solves "memory churn / forced update at every thinking step" in looped latent architectures. **Synthesis**: The strongest recent signal (MELT 2026) is that for looped/latent recurrent reasoning, the winning pattern is **decoupling the memory state evolution from the tight per-step recurrence**. G-MemLLM/LM2 add concrete gated bank implementations. Titans shows that utility/surprise (not opportunity frequency) is a powerful write trigger when placed at the right level of abstraction.

### 2. Mapping to Our Exact Failure (Humanistic + Causal)

**Current topology (embedded per-block slots)**:
- Every time the hybrid block runs one micro-step of thinking inside answer_state_loop, the SparseSlotRouter gets to read + potentially write.
- Write opportunity is almost continuous.
- The training signal "did this specific write improve the final answer quality many steps later?" is extremely weak and noisy.
- Result: Model learns excellent persistence (the notebook never falls apart) but almost no selective utility (removing the notebook doesn't hurt).

**What the papers suggest instead**:
- Memory lives as a somewhat slower, separate "bank" or evolving cache.
- The recurrent thinker (our OneBodyParallelHybridBlock) can **query** it when it needs context.
- A controller (gating, surprise, or learned policy) decides write timing and strength at a coarser granularity (after a burst of thinking, or during explicit rehearsal with clear utility signal).
- This gives the learning signal a chance to see "when I chose to write this, the later answer was better/worse."

This matches the project's own earlier diagnosis in the Big-Jump Assessment sections.

### 3. Proposed Architecture Candidate (Decoupled Latent Memory Bank)

**Name**: DecoupledLatentMemoryBank (MELT-inspired shared evolving bank + G-MemLLM/LM2-style explicit controller gates)

**High-level contract**:
- Separate module (not inside every hybrid block forward).
- `read(current_thought_state)` → rich context injected into the normal recurrent path (preserves One-Body).
- `controller_maybe_write(current_state, utility_or_surprise_signal, rehearsal_target)` → only called at meaningful moments (rehearsal steps, end of thinking bursts, or via learned halting).
- Strong persistence on non-written slots (preserve Raven/MSA inductive bias).
- Full ablation: `--use_decoupled_memory_bank` or equivalent flag + clean bank_off path (identical to current embedded behavior when disabled).

**Why this is a true Big Jump (per skill)**:
- Changes the causal route of memory participation from "synchronous side-effect of every recurrence step" → "explicit, utility-gated interaction between thinker and a separate memory resource".
- Directly attacks the "too many write opportunities → no utility learning" problem identified in all previous RI-4 rejects.

**Preservation commitments**:
- One-Body: Final answer still comes from the normal LM head after recurrent state (now potentially enriched by bank reads).
- Existing 5.56 rehearsal + RI-4 4-way contract remain executable (the new bank can be used orthogonally or as a replacement for the embedded router in future experiments).
- Reverse I→G→A for stochastic breadth and attractor protection.

## 2026-06 Radical Architecture Proposal: Latent Episode Memory (LEM) — True Causal Route Change

**User directive**: "완전 혁신적인 아이디어가 필요한거 아니야?" + "획기적인 방식으로 해결해"

After dozens of incremental and even "decoupled bank" experiments all returning the same negative signal (persistent_carry_rate = 1.0, no meaningful ablation effect), we accept that adding better memory mechanisms **on top of the current high-frequency tight recurrent loop** is still a local minimum.

### Humanistic Diagnosis (문과적)

The current student (OneBodyParallelHybridBlock + answer_state_loop) thinks in extremely fine-grained micro-steps (almost continuous delta updates).

We kept asking this student to be very selective about what to write in the notebook **every single micro-thought**.

Because write opportunities are almost continuous, the student never feels strong pressure to be picky. "I can always fix it in the next 0.1 second thought."

Even when we moved the bank outside the block (decoupled), as long as the controller was still being asked to decide writes at roughly the same high frequency during rehearsal, the fundamental problem remained.

### The Radical Shift: Episode-Based Memory Access

Instead of continuous or high-frequency memory interaction, we make memory access **sparse in thinking time**:

- The recurrence runs "episodes" (coherent chunks of micro-steps).
- During an active episode, the fast recurrent core operates mostly with transient state (memory influence is limited, e.g. only initial retrieval).
- Only at explicit **episode boundaries** (decided by a learned commit mechanism, surprise, uncertainty, or length) does the system produce a summary of the recent thinking trajectory and selectively write it into long-term persistent memory.
- Retrieval from long-term memory happens mainly at the start of new episodes or when the fast state signals high uncertainty.

This directly changes the causal frequency and nature of memory writes from "almost every thinking micro-step" to "only at the boundary of meaningful thought segments."

This is much closer to how biological systems separate fast thinking (working memory, transient) from slower selective consolidation (hippocampal → cortical transfer).

### Implementation Started (Minimal Prototype)

New module created:
`src/qtrm_mm/memory/latent_episode_memory.py`

Core components:
- `LatentEpisodeMemory` with long_term_bank + episode buffer
- `step_fast_state()` — called every micro-step to accumulate the episode trajectory (cheap)
- `commit_episode()` — the key selective write point (called at episode boundaries from trainer/rehearsal)
- `retrieve_at_episode_start()` — controlled retrieval at the beginning of new thinking segments
- Strong persistence on non-committed slots + explicit controllers for commit and retrieval

Basic hooks added to `OneBodyParallelHybridBlock` (step_fast_state accumulation when LEM is attached).

This is now the leading "획기적인" candidate direction.

The next smallest falsifying experiment would involve:
- Running longer continuations with explicit episode commit calls during 5.56 rehearsal (instead of per-step or per-rehearsal-step writes).
- Measuring whether the ablation signal on real heldout cases finally appears when memory writes are forced to be sparse in thinking time.

This is no longer "better memory on the old loop".
This is changing the loop + memory interaction contract at the architectural level.

### Four Distinct Directions to Test in Parallel

**Direction A: Change Recurrence Granularity Itself (Coarse Atomic Thought Steps)**
- Instead of every micro delta update being a "thinking step", introduce explicit coarser "thought steps" (multiple internal fast recurrence iterations that do not touch persistent memory).
- Memory read/write only allowed at the boundary of these coarser thought steps.
- Hypothesis: Increasing the "atomic unit of thinking" before memory access changes the learning dynamics more fundamentally than just changing write frequency.

**Direction B: Uncertainty-Gated Memory Access (Only Touch Memory When Confused)**
- Add an internal uncertainty estimator (e.g., prediction error, entropy proxy, or reconstruction error from a small auxiliary head).
- The recurrent core can only read from or commit to long-term memory when uncertainty exceeds a learned or scheduled threshold.
- Hypothesis: Forcing memory access to be strictly conditional on the model's own "I don't know" signal creates much stronger causal pressure for selective use.

**Direction C: Separate Consolidation Pass After Pure Recurrence Bursts**
- Run stretches of pure recurrence with memory completely disabled (or read-only with frozen bank).
- After N steps (or learned length), run a dedicated "consolidation pass" where the model explicitly decides what from the recent trajectory to write into persistent memory.
- This mimics biological offline consolidation more closely.
- Hypothesis: Separating "fast thinking" from "memory writing" in time and objective creates clearer gradients.

**Direction D: Limited-Bandwidth Global Workspace (Forced Selectivity Through Bottleneck)**
- Introduce a very small, learned "workspace" vector (e.g., 1-4 vectors of size d_model).
- The recurrent core must explicitly route / compress information through this workspace to reach long-term memory (or to retrieve from it).
- The workspace acts as a hard information bottleneck, forcing the model to learn what is worth keeping.
- Hypothesis: Architectural bottleneck is stronger than learned gating or timing for creating selectivity pressure.
