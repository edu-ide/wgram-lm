# Own-Latent Prediction Sample Complexity

Source:

- Korchinski, Favero, Wyart. "Learn from your own latents and not from
  tokens: A sample-complexity theory." arXiv:2605.27734, submitted
  2026-05-26. <https://arxiv.org/abs/2605.27734>

## Core Claim

The paper studies data generated by a hierarchical probabilistic grammar. In
that setting, objectives that supervise or reconstruct visible tokens need a
sample count that grows exponentially with latent-tree depth. Predicting the
model's own learned latent representations can recover the latent hierarchy
with sample count independent of tree depth up to logarithmic factors.

Plain-language read:

```text
Token prediction teaches the mouth.
Own-latent prediction teaches the hidden grammar.
```

This is a methodology source for QTRM/BLT, but it must not be read as "drop
token loss." A language model still needs a token-speaking path. The paper is
evidence that token loss alone may be a sample-inefficient way to discover the
compositional latent structure behind the token stream.

## Relevant Results For QTRM

| Paper result | QTRM interpretation |
|---|---|
| Token-level SSL and supervised token targets can be depth-expensive on hierarchical grammar data. | A pure byte/token CE run may plateau while still missing the latent parse/reasoning state. |
| Learning from own latents improves sample complexity on the analyzed hierarchy. | BLT/QTRM should have a same-body latent-prediction objective over internal selected-boundary/core states, not only next-byte CE. |
| data2vec is argued to perform implicit hierarchical latent prediction. | A single well-placed latent predictor may already induce hierarchy; do not add redundant stacked H-JEPA towers without an ablation. |
| Explicit stacking such as H-JEPA may be largely redundant in their setting. | Prefer one latent-prediction contract attached to the active one-body path before building separate multi-tower JEPA systems. |

## Boundary

This paper does not prove that:

- next-token or next-byte prediction can be removed from a language model,
- LeWM-style next-latent prediction alone will improve answer accuracy,
- an external world-model branch may bypass the one-body LM head,
- explicit memory/search/GRAM/PTRM are unnecessary.

It does support:

- training latent states to predict other same-model latent states,
- evaluating whether this improves data efficiency, depth scaling, and
  free-generation quality,
- treating token loss and latent-prediction loss as different roles rather
  than competing philosophies.

## Canonical Local Mapping

For active BLT/IMTA work, the compatible mapping is:

```text
DataIO bytes
-> BLT learned boundary states
-> recurrent global core states
-> same-body own-latent predictor
     predicts future / masked / adjacent internal latent states
     without becoming a separate answer speaker
-> hnet_causal_speaker
-> same LM head token answer
```

The predictor target should be an internal state that is already on the answer
causal path, for example selected-boundary hidden states, dechunked latent
states, or recurrent core states. It should not target a detached teacher state
unless that deviation is named and ablated.

## Required Ablations

Any promotion claim using this source needs:

- latent-prediction-off ablation,
- token-CE-only baseline under the same data/window/optimizer,
- same-mouth proof that the final answer still flows through the normal LM head,
- semantic/answer-causal gate, not only lower latent MSE,
- depth and K-scaling interaction check when combined with IMTA/GRAM/PTRM,
- free-generation loop and exact-answer gate.

## Conflict Resolution Rule

If a local document says "LeWM self-latent prediction failed," read that as:

```text
The previous implementation learned a latent transition that was not yet
semantic or answer-causal.
```

Do not read it as:

```text
Own-latent prediction is theoretically useless.
```

The 2026-05-26 paper strengthens the case for own-latent prediction as a
representation-learning objective, while preserving the existing QTRM rule that
the objective must prove semantic and answer-causal lift before entering the
canonical answer path.
