## Purpose

Before writing any integration code, we must have a clear, explicit map of:

- Where each critical 5.56 training dynamic lives in the **new architecture**.
- How it interacts with the Parallel Hybrid structure (recurrence branch with Gating v2/official GDN2 + attention branch with GQA/official MLA + vector fusion).
- Where the ablation hooks must be placed to preserve the Reverse I→G→A contracts.

This document is the design artifact for S1.1.

## 5.56 Core Dynamics (to be mapped)

1. **Scheduled External Binding Decay**
   - Historical: 0.40 → 0.04 over the curriculum
   - Controls how strongly the model is pulled toward the current rehearsal target

2. **Gold Structural Injection**
   - Injection of 642 gold state (bos_latent / attractor-baked direction) into the recurrent state
   - Modulated by current decay weight

3. **Attractor Protection During Rehearsal**
   - Historical value: 0.7
   - Prevents the attractor from being damaged while rehearsing important states

4. **Stochastic Recurrent Breadth**
   - K>1 noisy trajectories during recurrence (delta or true_gram mode)
   - Must remain ablatable with clean identity behavior

5. **Rehearsal Selection + Importance**
   - ALRMC-style importance scoring for which states to rehearse

## Target Architecture for Mapping

- `OneBodyParallelHybridBlock`
  - Recurrence branch: multiple heads (Gating v2 or official GDN2)
  - Attention branch: multiple heads (GQA or official MLA)
  - Vector gated fusion (per-dimension, with temperature)
  - Stochastic breadth injection point (currently after recurrence branch, before fusion in v0.2 design)

- `AdaptiveRehearsal` module (already exists)
- `QTRMConfig` + new hybrid-specific flags

## Current Hybrid Block Forward Structure (as of 2026-05-30)

```python
x_norm = norm1(x)

# Recurrence branch (multiple heads)
rec_outs = [head(x_norm) for head in recurrence_heads]
rec_projected = projection(concat(rec_outs))

# Stochastic breadth injection (current design)
if stochastic_breadth_noise and enabled and not ablation_zero:
    rec_projected = rec_projected + noise

# Attention branch
attn_outs = [attn(x_norm) for attn in attention_heads]
attn_projected = projection(concat(attn_outs))

# Vector gated fusion
fused = gate * rec_projected + (1-gate) * attn_projected

x = residual + fused
x = x + ffn(norm2(x))
return x
```

## Initial Mapping (Based on Current Code)

| 5.56 Dynamic                    | Recommended Injection Point                          | Rationale & Risks |
|--------------------------------|-------------------------------------------------------|-------------------|
| **Stochastic Breadth**         | After recurrence branch, before fusion (already partially implemented) | Must affect only recurrence side for clean ablation. Risk: fusion gate may partially cancel noise if not biased toward recurrence. |
| **Gold Structural Injection**  | Into `rec_projected` (before fusion) or into fused state | Gold should influence the long-term recurrent state. Injecting before fusion gives recurrence branch priority. |
| **Scheduled Binding Decay**    | Applied as a scalar multiplier on the rehearsal/gold pull term (outside the block, in rehearsal logic) | Decay is a training dynamic, not an architectural one. Should modulate how strongly we pull the fused (or post-fusion) state. |
| **Attractor Protection**       | During rehearsal update step (in `AdaptiveRehearsal`), protecting the state that is actually being updated (likely the fused output or recurrence branch) | Protection must be applied to whatever state the rehearsal is trying to improve. |
| **Rehearsal Importance**       | Computed on the fused state (or a combination of rec + attn) | The "importance" of a state should reflect the final representation after fusion. |
