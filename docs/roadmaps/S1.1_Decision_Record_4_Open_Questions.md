## The 4 Decisions + Final Ruling + Rationale

### 1. Stochastic Breadth Injection Scope
**Question**: Inject noise only into recurrence branch (pre-fusion) or also post-fusion / into fused state?

**Decision**: **Only into the recurrence branch, immediately after `rec_projected` and before fusion** (current v0.2 code location is correct and must be preserved).

**Justification**:
- Stochastic breadth is a *recurrent trajectory exploration* inductive bias. It belongs to the thinker (recurrence), not the final fused representation.
- Perfect ablation contract: when `core_stochastic_breadth_ablation_zero=True`, the caller passes None/zeros → rec_projected is untouched → after fusion the output must be numerically identical to the no-noise path (within floating point). Post-fusion injection would make this contract dirty or require complex compensation.
- Fusion gate (with recurrence bias init) is allowed to *modulate* how much of the noisy recurrence signal reaches the output, but the noise itself must originate from the recurrence branch so the ablation flag cleanly controls it.
- Risk if wrong: fusion gate could learn to cancel noise, silently destroying the 5.56 signal. Pre-fusion + recurrence bias init mitigates this.

**Implementation note**: Already wired in `OneBodyParallelHybridBlock.forward` (blocks.py:276-283). Do not move without new decision record.

### 2. Gold Structural Injection Target
**Question**: Inject 642 gold into recurrence branch input, fused output, or both?

**Decision**: **Primary injection into the recurrence branch (rec_projected or the hidden state feeding the recurrence heads), pre-fusion**.  
Light secondary pull on the *fused* state is allowed only during the explicit rehearsal update step (decay-modulated target pull).

**Justification**:
- Gold structural bias is a *long-term attractor / starting direction* for the recurrent memory. It must shape what the recurrence branch learns to carry across steps.
- Injecting only post-fusion would let the attention branch (short-term) dilute the gold signal before the recurrence branch ever sees it.
- Pre-fusion on recurrence gives the historical "642 gold + stochastic breadth" interaction the best chance to reproduce the strong diversity signal observed in the G-stage evidence (real gold amplified breadth benefit).
- Secondary post-fusion pull during rehearsal is acceptable because rehearsal is explicitly "improving the state that will be used next" — the fused state is the one being pulled toward the gold/rehearsal target.

**Risk control**: The recurrence branch must still receive a non-negligible fraction of the gold signal every step (controlled by the same decay schedule).

---

### 3. Attractor Protection Target During Rehearsal
**Question**: Which tensor do we protect (recurrence branch state, fused state, or both)?

**Decision**: **Protect the state that is actually being updated in the rehearsal step** (primary: the fused output state after the current block). The recurrence branch must inherit the protected direction through its next input (via residual + the fact that fused state carries the protected information forward).

**Justification**:
- Attractor protection (0.7 in historical recipe) prevents the important gold-baked direction from being overwritten during rehearsal of other material.
- The "state being rehearsed" in the new architecture is the one that will actually be fed to the next timestep's block — i.e., the output of the hybrid block (post-fusion).
- Protecting only the raw recurrence branch while the fusion gate can still heavily mix in unprotected attention signals would weaken the protection.
- However, the recurrence branch must not be *completely* bypassed; the protection logic should ensure that the recurrence heads continue to see and reinforce the protected attractor in their hidden dynamics.

**Practical rule**: In the rehearsal scheduler / curriculum step, apply the protection multiplier to the fused hidden state delta when computing the rehearsal target pull. The recurrence branch benefits automatically on the next forward because the protected fused state becomes (part of) its input.

---

### 4. Rehearsal Update Timing Relative to Fusion
**Question**: Should the decay-modulated gold/rehearsal target pull happen before fusion, after fusion, or inside the branches?

**Decision**: **The explicit rehearsal pull (pulling the current state toward gold/rehearsal target, modulated by scheduled decay) happens on the post-fusion fused state**.  
However, the recurrence branch must be given the *result* of that pull (or a strong fraction of it) as part of its input for the subsequent step. Gold injection (Decision 2) remains the main pre-fusion channel for structural bias.

**Justification**:
- Rehearsal is "make this important state better for future use." The state that will be used for future use is the fused output that exits the block.
- Doing the pull pre-fusion only on the recurrence branch would mean the attention branch can immediately dilute the just-rehearsed improvement on the very next step.
- Post-fusion pull + feeding the improved fused state back into the recurrence branch on the next block (via the normal residual stream) gives both branches the benefit of the rehearsal while preserving recurrence ownership of long-term dynamics.
- This matches the historical intent: the curriculum improves the quality of the recurrent state that the model actually relies on.

**One-Body check**: The final output state that produces logits / next predictions must be the one that received the rehearsal improvement. Post-fusion satisfies this.
