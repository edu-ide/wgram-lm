## Key Observation (S3 Diagnosis Trigger)

The critical Reverse I→G→A contract for stochastic recurrent breadth (the single strongest historical 5.56 inductive bias) is **not yet propagating cleanly** through the current OneBodyParallelHybridBlock + vector gated fusion.

Even when the block is constructed with `core_stochastic_breadth_ablation_zero=True` and the caller withholds noise, the downstream diversity proxy stays high.

This is **exactly the kind of signal S3 (Bottleneck Diagnosis) was designed to catch**.

Likely root causes (to be investigated in S3/S4):
- Vector fusion gate (even with recurrence bias init) is still allowing variance from other sources (gold injection simulation, random rehearsal target, etc.) to dominate the diversity metric.
- The simple diversity proxy used in the minimal script is not sufficiently isolated to the pre-fusion recurrence noise.
- Official MLA branch or the projection layers after concat are introducing additional variance that masks the ablation.
- The noise scale or injection point needs re-tuning once the metric is made more precise.

**Positive note**: The fact that we caught this on a 30-step smoke with the exact S1.1 decisions enforced is a win for the "순서대로 + contract first" discipline.
